body_en: "# Updated Wallarm Cybersecurity App\n\n_September 2018 – September 2019_  \nProduct designer\n\n\U0001FAE1Worked at Wallarm **from September 2018 to September 2019** product designer\n\n### Work process\n\nI started “debottling” the Wallarm product, a cybersecurity platform for protecting APIs (application programming interfaces) and web applications.\n\nRebuilt UX logic _(user experience),_ cleaned up the old UI _(user interface)_ and at the same time launched a full-fledged design system: a library of components + a system of semantic tokens, synchronized with the frontend.\n\n![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__01.png)\n\nIn cybersecurity, the key is to quickly distinguish noise from signal. I focused on the flow of incidents: from raw traffic and alerts to a clear picture of the attack, its priority and the next action. The interface has become “operator-like”: fewer clicks, better overview, clear statuses and response scenarios.\n\n![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__02.png)\n\n### What did you do about the product?\n\n  * **UX refactoring:** reworked navigation, triage and incident prioritization scenarios, assembled a single route “observation → incident → response.”\n  * **UI redesign:** unified tables, cards and event timelines, added skeletons, empty states and predictive tips.\n  * **Design system:** collected a library of components (tables with aggregations, filters, policy-editor, logs, diffs, YAML/JSON-viewer), described states and variations.\n  * **Semantic tokens:** colors, shadows, borders, typography, interactivity - through tokens exported to the front (CSS variables/TypeScript) so that design and code remain in sync.\n  * **Visual analytics:** readable graphics, status badges and risk marks to quickly see **What** happened and **what to do next**.\n\n![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__03.png)\n\n### Bottom line\n\nThe interface has become faster to use and clearer for security teams: less friction during triage, a smooth visual system, a common language of components between design and frontend.\n\nThe project turned disparate screens into a holistic tool that helps not just detect attacks, but manage the response.\n\n_Some more screens:_\n\n![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__04.png)![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__05.png)![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__06.png)![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__07.png)![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__08.png)![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__09.png)"
body_ru: "# Обновлял приложение Wallarm Cybersecurity App\n\n_Сентябрь 2018 — Сентябрь 2019_  \nПродуктовый дизайнер\n\n\U0001FAE1Работал в Wallarm **c сентября 2018 по сентябрь 2019 года** продуктовым дизайнером\n\n### Процесс работы\n\nЯ занялся «расшивкой узких мест» в продукте Wallarm — платформе кибербезопасности для защиты API (application programming interface) и веб-приложений.\n\nПересобрал логику UX _(user experience),_ навёл порядок в старом UI _(user interface)_ и параллельно запустил полноценную дизайн-систему: библиотеку компонентов + систему семантических токенов, синхронизированную с фронтендом.\n\n![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__01.png)\n\nВ кибербезопасности главное — быстро отличать шум от сигнала. Я сфокусировался на потоке инцидентов: от сырого трафика и алёртов к понятной картине атаки, её приоритету и следующему действию. Интерфейс стал «операторским»: меньше кликов, лучше обзор, ясные статусы и сценарии отклика.\n\n![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__02.png)\n\n### Что сделал по продукту\n\n  * **Рефакторинг UX:** переработал навигацию, сценарии триажа и приоритизации инцидентов, собрал единый маршрут «наблюдение → инцидент → ответ».\n  * **Редизайн UI:** унифицировал таблицы, карточки и таймлайны событий, добавил скелетоны, пустые состояния и предиктивные подсказки.\n  * **Дизайн-система:** собрал библиотеку компонентов (таблицы с агрегациями, фильтры, policy-editor, логи, диффы, YAML/JSON-viewer), описал состояния и вариативности.\n  * **Семантические токены:** цвета, тени, бордеры, типографика, интерактив — через токены, экспортируемые во фронт (CSS variables/TypeScript), чтобы дизайн и код оставались в синхроне.\n  * **Визуальная аналитика:** читабельные графики, статус-бэйджи и риск-метки, чтобы быстро видеть, **что** случилось и **что делать дальше**.\n\n![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__03.png)\n\n### Итог\n\nИнтерфейс стал быстрее в работе и понятнее для команд безопасности: меньше трения при триаже, ровная визуальная система, единый язык компонентов между дизайном и фронтендом.\n\nПроект превратил разрозненные экраны в целостный инструмент, который помогает не просто фиксировать атаки, а управлять ответом.\n\n_Еще некоторые экраны:_\n\n![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__04.png)![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__05.png)![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__06.png)![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__07.png)![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__08.png)![](https://cdn.nevsky.me/content/images/2026/03/wallarm-cybersecurity-app-cv__09.png)"
